# Example 5: `!status` With File Upload

Этот пример запускает бота с ответом по команде `!status`:
- бот работает с авто-подпиской на комнаты (или со статическим списком через `ROCKET_ROOM_IDS`);
- бот отвечает только на точную команду `!status`;
- бот отправляет файл в комнату через upload API;
- бот отвечает только разрешенному отправителю (`ROCKET_ALLOWED_SENDER_ID`).

Особенность этого примера: по команде `!status` бот загружает изображение (`UploadRoomFileDetailed`).

## Переменные окружения

- `ROCKET_SERVER_URL` - URL сервера Rocket.Chat, например `https://chat.example.com`
- `ROCKET_USER_ID` - ID пользователя бота
- `ROCKET_AUTH_TOKEN` - токен пользователя бота
- `ROCKET_ALLOWED_SENDER_ID` - ID пользователя, которому бот может отвечать
- `ROCKET_ROOM_IDS` - (опционально) список room ID через запятую, например `id1,id2,id3`
- `DRUM_IMAGE_PATH` - (опционально) путь к файлу для отправки; по умолчанию `drum.png`

## Что проверить после запуска

- В логах есть успешное подключение и подписка на комнаты.
- На сообщение `!status` от разрешенного пользователя бот отправляет файл.
- Сообщения от других пользователей игнорируются.
